# Анализ типа регистров

В рамках данного задания вы проанализируете тип регистра для данного вам исполнения.

## Инициализация

Для начала выполнения задания вам надо получить индивидуальный вариант задания.
Запустите `./gradlew assign`. Он прочитает имя вашего Git репозитория и создаст файл
[solution.txt](solution.txt) с вашим вариантом задания.

## Формат задания

В файл [solution.txt](solution.txt) будет записано 10 историй исполнения. В каждой истории
несколько строк. По строке на каждый поток исполнения (P, Q, ...). Например:

```
P: -----------[       R:1       ]-----[ W(2) ]-------
Q: -----[   R:0   ]----------------------------------
R: ----------[     W(1)    ]-------------------------
S: ---------------------------[   R:1   ]------------
```

В истории встречаются два типа операций:
* `W(x)` -- запись значения x в регистр.
* `R:x` -- чтения значения x из регистра.
В начале исполнения регистр инициализирован значением ноль.

Квадратные скобки `[` и `]` обозначают начало и конец операции.

## Задание

Определите тип регистра для каждого исполнения и запишите его перед каждой историей
вместо `<type>`. В этом задании регистр может как одновременно писаться, так и читаться из
нескольких потоков. При определении типа регистра, в котором есть несколько одновременных записей,
мы ищем полный порядок операций записи, сохраняющий отношение частичного порядка "произошло-до",
заданное исполнением, такой, что регистр имеет соответсвующую гарантию на чтение, как было
рассказано в лекции.

> То есть, например, в этом задании мы будем называть регистр "регулярным", если у него
> линеаризуемые записи и регулярное чтение.

Используйте следующие типы:

* `linearizable` -- если исполнение линеаризуемое, то есть регистр атомарный.
* `regular` -- если регистр имеет регулярное чтение. При чтении регулярного регистра гарантируется, что
  результат чтения будет равен либо результату последней завершенной записи, либо результату
  одной из операций записи которая происходит параллельно с этим чтением.
* `safe` -- если регистр имеет безопасное чтение. При чтении безопасного регистра гарантируется, что
  в случае отсутствия параллельных операций записей результат чтения будет равен результату последней
  завершенной записи. При чтении параллельно с операцией записи результат чтения может быть произвольным.
* `unsafe` -- если исполнение не безопасное, то есть не подходит даже под требование безопасного исполнения.

> Типы регистров даны в порядке уменьшения гарантий на чтение. Вы должны указать наиболее сильную
> (саму первую) гарантию, которой удовлетворяет исполнение.

## Проверка

Для проверки корректности вашего решения запустите `./gradlew build` из корня репозитория.

**Обратите внимание, что в случае ошибки следующая проверка вам будет доступа не раньше чем через 10 минут.**
При попытке проверить задание раньше, вам будет возвращена ошибка "Too Many Requests, retry after Xm YYs"
Перепроверяйте внимательно каждое исполнение. 

**Один раз вы можете попросить подсказку**. Для этого запустите `./gradlew build -Phint`. 
В этом случае вам будет указан один конкретный номер истории на котором у вас неправильный ответ 
(случайный из всех неправильных). Вы можете воспользоваться подсказкой только один раз, так что экономьте 
её на крайний случай, когда ошибку ни как не получается найти силой мысли.
